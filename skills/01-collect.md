# 01-collect — 知识采集 (Ingest)

> 从 get 笔记 API 批量抓取原始笔记，保存到本地 inbox。
>
> **执行频率**: 每日自动  
> **预期耗时**: 无新笔记 ~30 秒 | 5 条新笔记 ~1-2 分钟

---

## 能帮你做什么

- 从 get 笔记 API 增量抓取新笔记
- 自动跳过已处理的笔记（幂等性）
- 支持断点续传，中断后可恢复
- 使用早停策略，只抓取基准线之后的新内容
- 输出：`raw/notes/_inbox/*.md`

## 工作原理

```
┌─────────────────────────────────────────────────────────┐
│  知识采集 (Ingest)                                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  get 笔记 API                                            │
│      │                                                  │
│      ▼                                                  │
│  ┌─────────────────┐                                    │
│  │  早停遍历        │ ← 遇到旧笔记就停止分页               │
│  │  (Early Stop)   │                                    │
│  └─────────────────┘                                    │
│      │                                                  │
│      ▼                                                  │
│  raw/notes/_inbox/                                      │
│  (原始 Markdown)                                        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `batch_size` | 5 | 单批次最大处理数量 |
| `concurrency` | 1 | 并发请求数 |
| `rate_limit_delay` | 2.0s | API 请求间隔 |

---

## 基准线逻辑

**重要**：基准线 = 最新抓取笔记的 `created_at`（API 返回的笔记创建时间）

- 增量同步只抓取 `created_at > baseline` 的笔记
- 如果抓到 0 条，基准线**不更新**（避免遗漏中间笔记）
- 抓到新笔记后，基准线 = 最新笔记的 `created_at`

---

## 错误处理

| 错误 | 处理方式 |
|------|----------|
| API 限流 (429) | 自动等待，指数退避重试 |
| 网络超时 | 重试 3 次 |
| 基准线未设置 | 提示先执行全量同步 |
