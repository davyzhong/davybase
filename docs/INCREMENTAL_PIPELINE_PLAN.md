## 一、目标

完成 **10 条增量笔记** 的端到端处理测试，验证增量流水线的可行性和稳定性。

### 测试目标

1. ✅ 验证新增笔记检测机制
2. ✅ 验证 get 笔记 API 抽取流程
3. ✅ 验证 Worker 池消化模式 (小批量)
4. ✅ 验证分类归档逻辑
5. ✅ 验证幂等性 (重复运行不重复处理)

## 七、风险与缓解

| 风险 | 可能性 | 缓解措施 |
|------|--------|----------|
| get 笔记 API 限流 | 低 | 已有指数退避机制 |
| LLM 分类失败 | 中 | 复用全量消化的 fallback 逻辑 |
| 文件移动冲突 | 低 | 目标目录已存在则跳过 |
| sync.db 锁定 | 低 | 使用 WAL 模式 |
